# 冰鱼自动发布工具 - 详细设计文档

## 1. 数据库设计

### 1.1 数据库ER图设计

本系统数据库包含16张表，分为系统管理类和业务数据类两大类。系统管理类表用于实现用户认证、权限管理、日志记录等功能；业务数据类表用于存储产品、图片、关键词、模板、发布记录等业务数据。各表之间通过外键关联，形成完整的数据关系网络。

## 2. API接口设计

### 2.1 接口规范

API接口采用RESTful风格设计，使用JSON格式进行数据交互。接口基础路径为/api，认证相关接口路径为/api/auth，用户管理接口路径为/api/users，系统管理接口路径为/api/admin，业务接口根据资源类型命名。

所有接口的请求和响应均使用JSON格式。请求Content-Type为application/json，响应Content-Type为application/json。成功响应的标准格式为{"code": 200, "msg": "success", "data": {...}}，失败响应的格式为{"code": 400, "msg": "error message", "data": null}。需要认证的接口通过请求头Authorization传递JWT令牌，格式为"Bearer {token}"。

### 2.2 认证接口

#### 2.2.1 用户登录

接口地址：POST /api/auth/login

请求参数：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码（MD5加密） |

响应示例：

```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userInfo": {
      "userId": 1,
      "username": "admin",
      "nickname": "管理员",
      "role": "admin",
      "avatar": "https://xxx.com/avatar.png"
    }
  }
}
```

#### 2.2.2 用户登出

接口地址：POST /api/auth/logout

请求头：Authorization: Bearer {token}

响应示例：

```json
{
  "code": 200,
  "msg": "登出成功",
  "data": null
}
```

## 3. 后端业务逻辑设计

### 3.1 认证模块

认证模块负责用户登录登出和令牌管理。用户登录时，验证用户名密码是否正确，密码使用MD5加密后比对。验证成功后生成JWT令牌，令牌包含用户ID、用户名、角色信息，有效期24小时。令牌使用HS256算法签名，密钥从环境变量JWT_SECRET_KEY获取。登录日志记录用户名、IP地址、登录结果等信息。用户登出时，使当前令牌失效，后续请求将无法通过验证。

### 3.2 权限模块

权限模块基于RBAC（基于角色的访问控制）模型实现。用户拥有角色，角色拥有菜单权限。用户登录后获取用户信息和角色菜单权限，前端根据权限控制菜单和按钮的显示。用户访问接口时，后端验证用户角色是否有权访问该接口。

### 3.3 AI服务模块

AI服务模块负责调用第三方AI API生成文章内容。模块支持OpenAI和智谱AI两种服务商，通过配置不同的API地址和模型名称进行切换。生成内容时，系统根据选择的模板构造提示词，将产品信息、关键词等变量替换到提示词中，发送给AI API。AI返回内容后，系统解析返回结果，提取标题和正文内容。如果配置了图片，系统根据图片的位置设置将图片插入到内容适当位置。

### 3.4 发布服务模块

发布服务模块负责将文章发布到目标平台。模块使用Playwright进行浏览器自动化操作。首先加载网站配置中的Cookie，模拟用户登录状态。然后导航到发布页面，填写文章标题和内容。最后点击发布按钮提交文章。发布过程中实时记录发布状态，发布完成后更新记录状态和返回链接。

## 4. 前端组件设计

### 4.1 布局组件

#### 4.1.1 主布局组件（MainLayout）

主布局组件包含顶部导航栏、侧边栏和主要内容区域三部分。顶部导航栏显示系统名称、当前用户信息、退出登录按钮。侧边栏显示导航菜单，支持折叠展开。主要内容区域显示页面内容，使用router-view渲染。

#### 4.1.2 侧边栏组件（Sidebar）

侧边栏组件根据用户权限动态生成菜单项。每个菜单项包含图标和名称，支持多级菜单。菜单数据从后端获取，存储到Pinia中。点击菜单项跳转到对应页面，当前选中菜单高亮显示。

### 4.2 通用组件

#### 4.2.1 表格组件（Table）

表格组件封装Element Plus的el-table组件，提供统一的数据展示和操作功能。组件支持列配置、排序、分页、选择等功能。操作列显示编辑、删除等按钮，按钮根据权限控制显示。

#### 4.2.2 表单组件（Form）

表单组件封装Element Plus的el-form组件，提供统一的数据录入功能。组件支持多种表单项类型，包括输入框、选择框、日期选择器、文件上传等。组件内置校验规则，提交前自动验证数据有效性。

#### 4.2.3 对话框组件（Dialog）

对话框组件封装Element Plus的el-dialog组件，提供弹窗功能。组件支持标题、内容、底部按钮的自定义配置。打开对话框时传递数据，关闭时返回数据。

### 4.3 业务组件

#### 4.3.1 产品选择器

产品选择器组件用于选择推广产品。组件展示产品列表，支持搜索和分页。选择产品后触发change事件，返回产品ID和名称。

#### 4.3.2 网站复选框组

网站复选框组组件用于选择目标发布网站。组件展示已配置的可发布网站列表，支持多选。选择网站后返回网站ID列表。

文章编辑器组件用于编辑生成的文章内容。组件支持富文本编辑，提供基本的格式工具。组件可嵌入图片，显示图片插入位置标记。

## 5. 核心流程设计

### 5.1 文章生成流程

文章生成流程如下：用户选择产品、网站、模板和AI模型后，点击生成按钮触发生成流程。首先系统获取模板配置和关键词列表，将产品信息、关键词等填充到提示词模板中。然后调用AI服务API发送生成请求，AI返回生成的标题和内容。接着系统处理返回内容，根据图片配置将图片插入到内容适当位置。最后返回生成结果供用户确认和编辑。

### 5.2 文章发布流程

文章发布流程如下：用户确认文章内容后，点击发布按钮触发发布流程。系统创建发布任务，生成任务ID。遍历目标网站列表，为每个网站创建发布子任务。使用Playwright加载网站Cookie，模拟登录状态。导航到发布页面，填写标题和内容，点击发布按钮提交。发布完成后更新记录状态和返回链接。实时推送发布进度到前端，前端通过轮询或WebSocket获取进度更新。

### 5.3 批量导入流程

批量导入流程用于快速导入关键词。用户上传Excel文件后，前端将文件发送到后端。后端使用openpyxl库解析Excel文件，读取关键词列数据。逐条创建关键词记录，返回导入结果。前端显示导入成功和失败的数量，失败时显示失败原因。
