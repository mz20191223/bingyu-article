# 提示词模板整合需求文档

## 1. 需求背景

当前系统存在**标题模板**和**内容模板**两个独立的模板管理模块，使用时需要分别配置和调用。为简化操作流程，提高生成效率，用户希望将两者合并为一个统一的**提示词模板**，并按照指定格式输出推广软文。

**核心点**：提示词模板由用户在后台配置，系统负责存储和调用AI生成，不做硬编码限制。

## 2. 核心需求

### 2.1 模板合并
- 将现有的「标题模板」和「内容模板」合并为一个统一的「提示词模板」
- 模板需关联产品（多对多关系保持不变）
- 模板需包含完整的提示词配置，指导AI生成符合格式要求的推广软文

### 2.2 关键词管理
- 关键词是全局共享的，不与产品关联
- 在生成内容时由用户手动选择关键词，或随机抽取
- 支持从「关键词管理」模块选择多个关键词
- 关键词将嵌入提示词中，指导AI生成相关性更高的内容

### 2.3 输出格式规范（必须严格遵守）

#### 格式要求
- 第一行必须是标题，标题要吸引人、有吸引力，不要用【】包裹
- 第二行开始是正文
- 每个段落必须用【】包裹小标题，小标题单独占一行
- 小标题下面是段落内容，段落内容不要换行
- 段落与段落之间必须用两个换行分隔（空一行）
- 不要使用任何Markdown格式（如#、*、>等）
- 不要使用HTML标签
- 纯文本格式输出，至少写5个正文段落（不含标题）

#### 内容要求
- 语言亲切自然，口语化
- 突出产品的核心优势
- 包含用户痛点和解决方案
- 绝对不能提到任何其他竞品APP的名称
- 绝对不能输出任何其他人的邀请码、微信号、联系方式或推广信息

## 3. 功能模块设计

### 3.1 提示词模板管理
- **新增模板**：创建统一的提示词模板，配置完整的AI生成规则
- **编辑模板**：修改模板内容和配置
- **删除模板**：删除不需要的模板
- **关联产品**：支持为模板关联多个产品（多选）
- **设置默认**：设置默认模板

### 3.2 AI生成接口
- **单篇生成**：根据产品和模板生成单篇推广软文
- **批量生成**：支持为多个产品批量生成软文
- **预览功能**：生成后可预览内容

### 3.3 关键词集成
- **手动选择**：生成时由用户从关键词库中选择
- **随机抽取**：支持系统随机抽取关键词
- **多选支持**：支持同时选择多个关键词
- **嵌入提示词**：选中的关键词将自动嵌入提示词中

## 4. 数据结构设计

### 4.1 提示词模板表（prompt_templates）

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | Integer | 主键ID |
| name | String | 模板名称 |
| prompt_content | Text | 完整的提示词内容 |
| required_paragraphs | Integer | 要求的段落数量（默认5） |
| conclusion_text | Text | 结尾引导固定内容 |
| business_type | String | 业务类型 |
| is_default | Integer | 是否默认（0/1） |
| status | Integer | 状态（0启用/1禁用） |
| create_time | DateTime | 创建时间 |
| update_time | DateTime | 更新时间 |

### 4.2 模板产品关联表（template_product）

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| template_id | Integer | 模板ID（外键） |
| product_id | Integer | 产品ID（外键） |

## 5. API接口设计

### 5.1 模板管理接口

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/prompt-templates` | GET | 获取模板列表 |
| `/api/prompt-templates` | POST | 创建模板 |
| `/api/prompt-templates/{id}` | PUT | 更新模板 |
| `/api/prompt-templates/{id}` | DELETE | 删除模板 |
| `/api/prompt-templates/{id}/default` | PUT | 设置默认模板 |

### 5.2 AI生成接口

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/ai/generate` | POST | 生成推广软文 |

#### 请求参数
```json
{
    "product_id": 1,
    "template_id": 1,
    "keywords": ["高省返利APP", "省钱"]
}
```

#### 响应示例
```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "title": "每次网购都在白白多花钱，你心疼不心疼",
        "content": "【为什么大家都在找省钱工具】\n现在很多人网购都想省钱，但优惠券太分散，找起来很麻烦。\n\n【高省返利APP的优势】\n高省返利APP整合了各大平台的优惠券，一键搜索就能找到最优惠的价格。\n...",
        "full_text": "每次网购都在白白多花钱，你心疼不心疼\n\n【为什么大家都在找省钱工具】\n..."
    }
}
```

#### 内容解析逻辑
AI返回的内容按以下规则解析分离：
1. **标题提取**：按换行符分割内容，第一行即为标题
2. **正文保留**：剩余内容为正文，保留原始【小标题】格式
3. **完整性校验**：确保包含至少5个段落和结尾引导

## 6. 前端页面设计

### 6.1 提示词模板列表页
- 展示模板列表（名称、关联产品、状态）
- 支持新增、编辑、删除、设为默认操作
- 表格列：ID、模板名称、关联产品、业务类型、默认、状态、操作

### 6.2 模板编辑弹窗
- 模板名称输入框
- 提示词内容文本域（多行）
- 段落数量选择（默认5）
- 结尾引导文本域（固定内容）
- 产品多选下拉框
- 状态开关

### 6.3 AI生成页面（集成到文章发布页）
- 产品选择下拉框
- 模板选择下拉框（根据产品关联自动筛选）
- 关键词输入区：
  - 支持手动输入关键词（逗号分隔）
  - 支持从关键词库多选
  - 支持「随机抽取」按钮，一键随机获取1个关键词
- 生成按钮
- 预览区域（展示生成的完整软文）
- **自动回填**：生成成功后，自动将标题填入标题输入框，正文填入内容编辑器

## 7. 业务流程

```
用户选择产品 → 选择/自动匹配模板 → 选择关键词 → 点击生成 → AI返回结果 → 预览/编辑 → 发布
```

## 8. 注意事项

1. **格式严格性**：AI生成结果必须严格符合指定格式，否则需要重新生成
2. **竞品过滤**：需要对生成内容进行校验，确保不包含竞品信息
3. **推广信息保护**：结尾引导信息是用户专属，需确保不会被AI修改或替换
4. **关键词使用**：用户需在生成时手动选择关键词，关键词全局共享
5. **段落数量**：确保至少生成5个正文段落
6. **标题正文分离**：AI返回内容后，系统自动解析分离标题和正文，支持回填到表单
