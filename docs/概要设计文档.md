# 软文自动发布工具 - 概要设计文档

## 1. 项目概述

### 1.1 项目背景

随着内容营销的快速发展，企业需要同时在多个内容平台发布推广文章来获取流量。人工逐一发布文章效率低下，且难以统一管理。本项目旨在构建一套基于AI的多平台文章自动生成与发布管理系统，通过AI技术自动生成高质量的软文内容，并支持一键发布到多个内容平台，大幅提升运营效率。

### 1.2 项目目标

本项目旨在实现以下核心目标：构建一套完整的软文自动发布系统，支持AI自动生成文章标题和内容，支持多平台（YOMO、UUUHAO、HXM18）一键发布，提供完善的后台管理功能，包括账号管理、产品管理、图片管理、关键词管理、模板管理、发布记录追踪等模块。系统需具备良好的用户体验和可扩展性，能够满足企业内容营销的日常工作需求。

### 1.3 项目范围

本项目的建设范围涵盖以下内容：后端API服务建设，采用Flask框架构建RESTful API，提供完整的业务逻辑支撑；前端Web应用开发，采用Vue 3 + Element Plus构建用户界面，提供友好的操作体验；数据库设计与实现，基于MySQL设计并实现完整的数据库结构；AI集成模块，实现与多种AI模型的对接，支持文章自动生成；平台发布模块，基于Playwright实现浏览器自动化，支持多平台文章发布。

## 2. 技术架构

### 2.1 整体架构

本项目采用前后端分离的架构模式，后端使用Flask框架提供RESTful API服务，前端使用Vue 3 + Element Plus构建单页应用。数据库采用MySQL，存储业务数据和配置信息。AI服务通过HTTP调用第三方API实现，网站发布通过Playwright浏览器自动化实现。

```
┌─────────────────────────────────────────────────────────────────┐
│                        前端 (Vue 3 + Element Plus)               │
│                    运行于浏览器，提供用户界面                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼ HTTP/REST API
┌─────────────────────────────────────────────────────────────────┐
│                        后端 (Flask)                               │
│              提供业务逻辑处理、数据存储、第三方服务集成            │
└─────────────────────────────────────────────────────────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
    ┌───────────┐       ┌───────────┐       ┌───────────┐
    │  MySQL    │       │  AI 服务  │       │  网站平台 │
    │  数据库   │       │(OpenAI等)  │       │(YOMO等)   │
    └───────────┘       └───────────┘       └───────────┘
```

### 2.2 技术选型

#### 后端技术栈

后端开发采用Python语言，使用Flask作为Web框架。Flask是一个轻量级的Web框架，灵活度高，易于扩展，非常适合构建中小型的RESTful API服务。数据库采用MySQL 5.7及以上版本，MySQL是成熟的关系型数据库，能够满足业务数据的存储和查询需求。AI集成使用openai库和zhipuai库，分别支持OpenAI GPT系列模型和智谱AI模型。浏览器自动化使用Playwright库，提供可靠的网页自动化操作能力。数据库连接使用mysql-connector-python驱动。

#### 前端技术栈

前端开发采用Vue 3作为核心框架，使用Composition API进行组件开发。UI组件库选用Element Plus，提供丰富的企业级组件。路由管理使用Vue Router 4，实现单页应用的路由控制。状态管理使用Pinia，替代Vuex提供更简洁的状态管理方案。HTTP客户端使用Axios，与后端API进行数据交互。

## 3. 功能模块设计

### 3.1 模块概述

本系统根据业务需求划分为十大功能模块：账号管理模块负责用户认证和权限控制；推广产品管理模块维护待推广的产品信息；图片管理模块管理文章配图资源；关键词管理模块维护AI生成文章用的关键词库；内容生成模板模块配置AI生成内容的提示词模板；标题生成模板模块配置AI生成标题的提示词模板；AI模型配置模块管理第三方AI API连接信息；网站配置模块管理发布目标平台的登录信息；文章发布模块是核心业务模块，负责文章生成和发布；发布记录模块追踪所有发布历史；日志管理模块记录系统操作行为。

### 3.2 账号管理模块

账号管理模块是系统的基础模块，负责用户认证和权限控制。该模块包含用户管理、角色管理、部门管理三个子模块。用户管理实现用户的增删改查功能，密码采用MD5加密存储，支持为用户分配角色。角色管理预定义了超级管理员、运营、访客三种角色，不同角色拥有不同的功能权限。部门管理支持组织架构的维护，用于多部门协同使用场景。用户登录采用用户名密码方式，登录成功后返回JWT令牌，后续请求通过令牌进行身份认证。

### 3.3 推广产品管理模块

推广产品管理模块用于维护待推广的产品信息。产品信息包括产品名称、产品链接、产品状态等字段。产品列表支持分页展示和名称搜索功能，新增产品时需填写产品名称和产品链接。编辑产品时可修改产品名称和链接，删除产品采用逻辑删除方式，仅标记删除状态而非物理删除。产品状态用于控制产品是否参与发布，可随时启用或禁用。

### 3.4 图片管理模块

图片管理模块用于管理文章配图资源，支持段落级别的图片插入位置设置。每张图片记录其URL地址、插入位置、适用网站、关联产品等信息。插入位置支持四种模式：自动模式由系统根据内容智能插入、第N段模式指定具体段落编号、首段模式固定插入文章开头、尾段模式固定插入文章结尾。图片可关联到指定产品，发布时自动匹配关联产品的文章。图片支持启用禁用控制，禁用的图片不会参与发布。

### 3.5 关键词管理模块

关键词管理模块维护AI生成文章用的关键词库。关键词是AI生成内容的核心素材，AI会根据选中的关键词生成符合营销需求的文章。关键词支持增删改查和批量导入功能。批量导入支持Excel格式，用户上传Excel文件后系统自动解析关键词并批量创建。关键词同样支持启用禁用控制，禁用的关键词不会参与AI生成。

### 3.6 内容生成模板模块

内容生成模板模块配置AI生成正文内容的提示词模板。该模块包含内容生成模板和标题生成模板两个子模块。内容生成模板包含模板名称、业务类型、Meta信息、关键词提示词、内容提示词、结论提示词等配置项。提示词中可以使用占位符如{product_name}、{keywords}等，系统会在生成时自动替换为实际值。每个模板可关联到指定产品，可设为默认模板。标题生成模板结构类似，但只包含标题生成提示词配置。

### 3.7 AI模型配置模块

AI模型配置模块管理第三方AI API的连接信息。支持的AI服务提供商包括OpenAI GPT系列和智谱AI。每条模型配置记录包含模型名称、API Key、API地址、调用参数等字段。系统支持配置多个AI模型，用户可在发布时选择使用哪个模型。可设置默认模型，生成文章时自动使用默认模型。模型配置支持增删改查和设为默认功能。

### 3.8 网站配置模块

网站配置模块管理目标发布平台的登录信息。当前支持YOMO、UUUHAO、HXM18三个平台。每个网站配置包含网站名称、登录URL、发布URL、Cookie等信息。Cookie是发布文章的关键凭证，用户需手动获取并配置。网站配置支持增删改查功能，删除时仅标记删除状态。系统使用Playwright加载Cookie后模拟登录，进而完成文章发布操作。

### 3.9 文章发布模块

文章发布模块是系统的核心业务模块，负责将AI生成的文章发布到目标平台。该模块的工作流程如下：首先用户选择要推广的产品，然后选择目标发布网站（可多选），接着选择内容生成模板和标题生成模板。点击生成按钮后，系统调用AI服务自动生成标题和内容。用户可手动编辑生成的标题和内容进行微调。系统根据图片配置自动将图片插入到文章适当位置。最后点击发布按钮，系统同时向选中的多个网站发布文章，并实时显示各网站的发布进度和状态。

### 3.10 发布记录模块

发布记录模块追踪所有文章发布历史。每条发布记录包含发布时间、发布账号、目标网站、文章标题、发布状态、返回链接等信息。发布状态分为待发布、发布中、发布成功、发布失败四种。记录列表支持按网站、状态进行筛选过滤。用户可查看每条记录的详细发布信息，也可删除单条记录或清空所有记录。

### 3.11 日志管理模块

日志管理模块记录系统运行过程中的关键操作行为。系统记录两类日志：操作日志记录用户的增删改查等业务操作行为，包括操作人、操作时间、操作内容、操作IP等信息；登录日志记录用户的登录登出行为，包括登录账号、登录时间、登录IP、登录结果等信息。日志支持按类型、时间范围进行查询，默认展示最近一个月的日志记录。

## 4. 数据库设计

### 4.1 数据库总体设计

本系统使用MySQL数据库，根据业务需求设计16张数据表。系统管理相关表包括sys_user（用户表）、sys_role（角色表）、sys_menu（菜单表）、sys_dept（部门表）、sys_post（岗位表）、sys_user_role（用户角色关联表）、sys_role_menu（角色菜单关联表）、sys_oper_log（操作日志表）、sys_login_log（登录日志表）。业务数据相关表包括products（推广产品表）、images（图片库表）、keywords（关键词库表）、content_prompt_templates（内容生成模板表）、title_prompt_templates（标题生成模板表）、websites（网站配置表）、models（AI模型配置表）、publish_records（发布记录表）。

### 4.2 核心表结构

#### 4.2.1 用户表（sys_user）

用户表存储系统用户信息，主要字段包括：用户ID（主键自增）、用户名（唯一索引）、密码（MD5加密）、昵称、手机号、邮箱、头像、状态（0正常1禁用）、部门ID（外键关联sys_dept）、岗位ID（外键关联sys_post）、备注、创建时间、更新时间。

#### 4.2.2 角色表（sys_role）

角色表存储系统角色信息，主要字段包括：角色ID（主键自增）、角色名称、角色编码（如admin、operator、guest）、显示顺序、状态（0正常1禁用）、备注、创建时间、更新时间。

#### 4.2.3 推广产品表（products）

推广产品表存储待推广的产品信息，主要字段包括：产品ID（主键自增）、产品名称、产品链接、产品状态（0正常1禁用）、删除状态（0未删除1已删除）、创建者ID、创建时间、更新时间。

#### 4.2.4 图片库表（images）

图片库表存储文章配图信息，主要字段包括：图片ID（主键自增）、图片URL、插入位置类型（auto/first/last/custom）、自定义段落号、适用网站（JSON数组）、关联产品ID、图片状态（0正常1禁用）、删除状态、创建者ID、创建时间、更新时间。

#### 4.2.5 关键词表（keywords）

关键词表存储关键词信息，主要字段包括：关键词ID（主键自增）、关键词内容、关键词状态（0正常1禁用）、删除状态、创建者ID、创建时间、更新时间。

#### 4.2.6 内容生成模板表（content_prompt_templates）

内容生成模板表存储AI内容生成提示词模板，主要字段包括：模板ID（主键自增）、模板名称、业务类型、Meta标题、Meta描述、关键词提示词、内容提示词、结论提示词、关联产品ID、是否默认（0否1是）、模板状态（0正常1禁用）、删除状态、创建者ID、创建时间、更新时间。

#### 4.2.7 标题生成模板表（title_prompt_templates）

标题生成模板表存储AI标题生成提示词模板，主要字段包括：模板ID（主键自增）、模板名称、业务类型、标题提示词、关联产品ID、是否默认（0否1是）、模板状态（0正常1禁用）、删除状态、创建者ID、创建时间、更新时间。

#### 4.2.8 AI模型配置表（models）

AI模型配置表存储AI API配置信息，主要字段包括：模型ID（主键自增）、模型名称、服务商类型（openai/zhipuai）、API Key、API地址、默认模型名称、调用参数（JSON）、是否默认（0否1是）、状态（0正常1禁用）、创建者ID、创建时间、更新时间。

#### 4.2.9 网站配置表（websites）

网站配置表存储发布目标平台信息，主要字段包括：网站ID（主键自增）、网站名称、网站标识（如yomo/uuuhao/hxm18）、登录URL、发布URL、Cookie（加密存储）、网站状态（0正常1禁用）、删除状态、创建者ID、创建时间、更新时间。

#### 4.2.10 发布记录表（publish_records）

发布记录表存储文章发布历史，主要字段包括：记录ID（主键自增）、关联产品ID、目标网站ID、AI模型ID、使用的标题模板ID、使用的内容模板ID、文章标题、文章内容（JSON，含图片插入信息）、发布状态（pending/publishing/success/failed）、返回链接、失败原因、发布者ID、发布时间、创建时间。

## 5. API设计

### 5.1 API总体设计

后端API采用RESTful风格设计，使用JSON格式进行数据交互。API基础路径为/api，认证接口路径为/api/auth，管理类接口路径为/api/admin，业务类接口路径根据资源类型命名。所有需要认证的接口通过请求头Authorization传递JWT令牌。

### 5.2 认证接口

认证接口包括用户登录和令牌刷新两个接口。登录接口POST /api/auth/login接收用户名和密码（MD5加密），验证成功后返回JWT令牌和用户信息。登出接口POST /api/auth/logout用于清除会话。刷新接口POST /api/auth/refresh用于刷新令牌有效期。

### 5.3 用户管理接口

用户管理接口包括用户的增删改查和密码修改。获取用户列表接口GET /api/users支持分页、名称搜索、条件筛选。获取单个用户接口GET /api/users/:id。新增用户接口POST /api/users。更新用户接口PUT /api/users/:id。删除用户接口DELETE /api/users/:id。修改密码接口PUT /api/users/:id/password。

### 5.4 产品管理接口

产品管理接口包括产品的增删改查和状态切换。获取产品列表接口GET /api/products支持分页、名称搜索。获取单个产品接口GET /api/products/:id。新增产品接口POST /api/products。更新产品接口PUT /api/products/:id。删除产品接口DELETE /api/products/:id（逻辑删除）。启用禁用接口PUT /api/products/:id/status。

### 5.5 图片管理接口

图片管理接口包括图片的增删改查和批量操作。获取图片列表接口GET /api/images支持分页、产品筛选、网站筛选、状态筛选。获取单张图片接口GET /api/images/:id。新增图片接口POST /api/images。更新图片接口PUT /api/images/:id。删除图片接口DELETE /api/images/:id。启用禁用接口PUT /api/images/:id/status。

### 5.6 关键词管理接口

关键词管理接口包括关键词的增删改查和批量导入。获取关键词列表接口GET /api/keywords支持分页、状态筛选。新增关键词接口POST /api/keywords。更新关键词接口PUT /api/keywords/:id。删除关键词接口DELETE /api/keywords/:id。启用禁用接口PUT /api/keywords/:id/status。批量导入接口POST /api/keywords/import接收Excel文件并批量创建关键词。

### 5.7 模板管理接口

模板管理接口包括内容模板和标题模板的增删改查。获取内容模板列表接口GET /api/content-templates支持分页、产品筛选、状态筛选。获取标题模板列表接口GET /api/title-templates支持分页、产品筛选、状态筛选。新增、编辑、删除接口分别对应POST、PUT、DELETE方法。设置默认接口PUT /api/content-templates/:id/default和PUT /api/title-templates/:id/default。

### 5.8 模型配置接口

模型配置接口包括AI模型配置的增删改查和默认设置。获取模型列表接口GET /api/models支持分页、状态筛选。获取单个模型接口GET /api/models/:id。新增模型接口POST /api/models。更新模型接口PUT /api/models/:id。删除模型接口DELETE /api/models/:id。设置默认接口PUT /api/models/:id/default。

### 5.9 网站配置接口

网站配置接口包括网站配置的增删改查。获取网站列表接口GET /api/websites支持分页、状态筛选。获取单个网站接口GET /api/websites/:id。新增网站接口POST /api/websites。更新网站接口PUT /api/websites/:id。删除网站接口DELETE /api/websites/:id（逻辑删除）。

### 5.10 发布接口

发布接口是核心业务接口，包括文章生成和发布。生成文章接口POST /api/publish/generate接收产品ID、网站ID、模板ID等参数，调用AI服务生成标题和内容，返回生成结果供用户确认。发布文章接口POST /api/publish/submit接收文章标题、内容、目标网站列表等参数，执行发布操作并返回发布进度。获取发布进度接口GET /api/publish/progress/:task_id用于轮询发布状态。

### 5.11 发布记录接口

发布记录接口包括记录查询和删除。获取记录列表接口GET /api/records支持分页、网站筛选、状态筛选、时间范围筛选。获取记录详情接口GET /api/records/:id。删除单条记录接口DELETE /api/records/:id。清空所有记录接口DELETE /api/records。

### 5.12 日志接口

日志接口包括操作日志和登录日志的查询。获取操作日志接口GET /api/logs/oper支持分页、操作类型筛选、时间范围筛选。获取登录日志接口GET /api/logs/login支持分页、时间范围筛选。

## 6. 前端架构

### 6.1 前端技术架构

前端采用Vue 3框架进行开发，使用Composition API编写组件。UI组件库选用Element Plus，该组件库提供了丰富的企业级UI组件，包括表格、表单、对话框、导航菜单等，能够快速构建功能完善的管理后台。前端路由使用Vue Router 4，通过路由守卫实现页面权限控制。状态管理使用Pinia，用于存储用户信息、权限信息、全局配置等数据。HTTP请求使用Axios，通过请求拦截器和响应拦截器统一处理认证令牌和错误信息。

### 6.2 页面结构

前端页面采用经典的管理后台布局：顶部Header区域显示系统名称、当前用户、退出登录按钮；左侧Sidebar区域显示导航菜单，支持折叠展开；中间Content区域显示主要内容。路由配置采用嵌套路由结构，根路由显示布局组件，子路由显示具体页面。

### 6.3 核心页面

前端核心页面包括：登录页面提供用户名密码输入和登录功能；仪表盘页面展示系统概览数据；用户管理页面管理用户账号；角色管理页面管理角色权限；产品管理页面管理推广产品；图片管理页面管理文章配图；关键词管理页面管理关键词库；内容模板页面管理内容生成模板；标题模板页面管理标题生成模板；模型配置页面管理AI模型；网站配置页面管理发布网站；文章发布页面是核心业务页面；发布记录页面查看发布历史；操作日志页面查看用户操作记录；登录日志页面查看登录记录。

### 6.4 权限控制

前端权限控制采用菜单级别和按钮级别双重控制。菜单权限由后端返回，用户只能看到有权限的菜单项。按钮权限通过自定义指令实现，控制页面的增删改按钮是否显示。用户登录后获取用户信息和权限菜单，存储到Pinia状态管理中，供各组件使用。

## 7. 安全设计

### 7.1 认证与授权

系统采用JWT令牌进行身份认证。用户登录成功后生成JWT令牌，有效期为24小时，令牌包含用户ID和角色信息。每次请求在请求头中携带令牌，后端验证令牌有效性后获取用户信息进行业务处理。密码采用MD5加密存储，传输过程使用HTTPS协议保护。

### 7.2 输入验证

所有用户输入均进行验证处理。后端使用参数校验库对请求参数进行验证，防止SQL注入、XSS攻击等安全风险。前端使用Element Plus表单组件的校验规则对输入进行验证，提供友好的错误提示。

### 7.3 敏感数据保护

系统中的敏感数据如API Key、Cookie等进行加密存储。使用Fernet对称加密算法对敏感字段进行加密，密钥通过环境变量配置，不存储在代码仓库中。

## 8. 部署方案

### 8.1 开发环境部署

开发环境部署较为简单，后端使用Flask内置开发服务器运行，直接执行python run.py即可启动服务。前端使用Vite开发服务器运行，执行npm run dev启动开发服务器，开发服务器支持热更新。

### 8.2 生产环境部署

生产环境建议使用Gunicorn运行Flask应用，配置Nginx作为反向代理和静态文件服务器。前端构建后生成静态文件，由Nginx提供访问。数据库使用MySQL生产实例，确保数据安全和性能。AI服务和网站发布服务可配置为后台运行或使用supervisor管理。

### 8.3 环境变量配置

系统通过环境变量配置敏感信息和运行环境参数。必须配置的环境变量包括数据库连接信息（DB_HOST、DB_PORT、DB_NAME、DB_USER、DB_PASSWORD）、JWT密钥（JWT_SECRET_KEY）、加密密钥（ENCRYPT_KEY）、AI服务API Key（OPENAI_API_KEY、ZHIPU_API_KEY）等。
