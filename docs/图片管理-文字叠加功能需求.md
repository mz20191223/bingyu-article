# 图片管理 - 图片合成文字功能需求

## 一、需求背景

用户在推广文章时，需要在图片上添加邀请码、联系方式等文字信息。目前流程是手动编辑图片后上传，较为繁琐。希望在图片管理中增加文字叠加功能，系统自动合成图片。

## 二、功能目标

在图片管理页面，上传图片后支持在图片指定位置添加文字，自动合成后保存。

## 三、功能详细设计

### 3.1 图片上传

- 支持上传 .png、.jpg、.jpeg 格式
- 支持本地上传，自动保存到服务器
- 上传后显示原图预览

### 3.2 文字叠加配置

| 配置项 | 选项/范围 | 默认值 |
|--------|----------|--------|
| 文字内容 | 用户输入（必填） | 空 |
| 字体大小 | 小(16px) / 中(24px) / 大(36px) | 中(24px) |
| 文字颜色 | 白色 / 黑色 / 红色 / 蓝色 | 白色 |
| 背景样式 | 无背景 / 半透明黑色 / 全透明黑色 | 无背景 |
| 定位方式 | 可拖拽定位（直接在图片上拖动文字到目标位置） | 底部居中 |
| 粗体 | 是 / 否 | 否 |

### 3.3 交互流程

1. 点击上传图片或拖拽图片到上传区域
2. 上传成功后显示原图预览
3. 输入要叠加的文字内容
4. 选择字体大小、颜色、背景样式
5. 在预览图上直接拖动文字到目标位置
6. 点击"预览"查看合成效果
7. 确认后系统自动合成并保存图片
8. 合成后的图片文件名添加 `_with_text` 后缀，原图保留

### 3.5 预览与确认

- 预览：点击"预览"按钮在页面实时显示合成效果
- 确认：点击"确认合成"保存合成后的图片
- 取消：放弃当前操作，清空表单

## 四、技术实现

### 4.1 后端实现

- 使用 Pillow 库进行图片合成
- ImageDraw：绘制文字
- ImageFont：字体样式
- 文字位置根据用户拖拽位置（百分比坐标）计算
- 合成后保存到原图同一目录，文件名添加 `_with_text` 后缀

### 4.2 前端实现

- Element Plus 组件库
- 图片上传使用 el-upload
- 预览区域支持文字拖拽定位（mousedown/mousemove/mouseup）
- 实时预览使用 CSS 定位叠加文字

### 3.6 数据库变更

现有 `images` 表增加字段：
```sql
ALTER TABLE images ADD COLUMN text_overlay JSON COMMENT '文字叠加配置';
ALTER TABLE images ADD COLUMN composite_url VARCHAR(500) COMMENT '合成后的图片URL';
```

text_overlay 字段结构：
```json
{
  "enabled": true,
  "text": "邀请码：088886",
  "fontSize": "medium",
  "color": "white",
  "bgStyle": "none",
  "positionX": 50,
  "positionY": 90,
  "bold": false
}
```

- positionX/positionY：文字左上角在图片上的百分比位置（0-100）
- composite_url：合成后的图片URL，为空表示未合成

### 3.7 发布逻辑调整

发布文章时查找图片规则：

1. 优先使用 composite_url（有合成图则用合成图）
2. 如果 composite_url 为空，则使用原 url（无合成图则用原图）

这样设计的好处：原图可复用，合成图按需生成。
